Welcome to Database User Interactive Menu, this is a collection of Oracle database routine operational tasks. Source code can be downloaded from https://github.com/automationdba/automationdba.git
This application is quite useful for Oracle Production DBA during troubleshoot any database related issues. This Apps mainly design for Oracle RAC/Cluster databases. However, this is compatible for non-RAC/stand-alone Oracle databases too.

My main objective is to help other DBA by sharing my expeirence in the area of Production Oracle DBA. We mostly perform the same task regularly. Now a days, every end user expect immedate result/response from Databaes irrespecitve of data growh or bad application design and hence as a DBA we have challenge to provide immediate satisfactory workaround to application folks. As everyone knows, Database query tuning is one of the most challenging task for DBA. Also, Oracle documentaiton have own limitation in this area. Being as Production DBA, applicaiton team expect to provide solution of "Long running Query" of their routine Batch Job. We hae always challenge for "db contention" in RAC/cluster databases. If applicatoin design flow and db optimization is not as per standard then everyone have to pay the price specially for DBA. Applicaiton db query suddently change from good plan to bad plan. As this kind of task, someitmes we don't have immediate resolution and hence, we need to explore various options/workaround. In such kind of situation, we need to fetch dictionary tables related different views and need to analyze those data and based on that, need to provide solution. This solution not always fix the issues but we need to follow the defined workaround sequentially. We generally run the database dictionry related query from sqlplus or any other tools like OEM,toad etc. This is our routine activity. In this scenario, I used to go one step ahead by doing Automation of those tasks. With the same intention and objective, I have developed a applicaiton as dbawebmenu. This applicaton is collection of various routine database inbuilt query. All those query is outcome of my experience in this area. This application is developed in django with integration of python and sql query. We just need to access database remotely and perform various required task as per requirement.


Some of the key features...

  • Freeware Oracle database performance monitor application
  • Automation of 50+ database related daily required task
  • Minimal interaction with database
  • Report can be generated parallel
  • History execution already archived
  • Two level of authentcation - application and Database level
  • Enhancement is quite easy
  • Completely secure as Only Database monitoring task includes in this version.
  • No any additional database privileges required
  • Report session self kill if db response takes more time
  • Additional sql_id based monitor menu
